# Issues Directory

**Purpose:** Track specific bugs and unexpected behaviors

---

## Subdirectories

| Directory | Contents |
|-----------|----------|
| `open/` | Active issues under investigation |
| `resolved/` | Fixed issues (archived for reference) |

## Issue Lifecycle

```
1. Create → open/ISSUE-XXX-Description.md
2. Investigate → Update status, add findings
3. Fix → Document resolution
4. Archive → Move to resolved/
```
