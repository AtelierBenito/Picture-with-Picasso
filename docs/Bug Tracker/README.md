# Bug Tracker

**Project:** Picture with Picasso
**Purpose:** Centralized issue and problem tracking

---

## Directory Structure

```
Bug Tracker/
    issues/
        open/           # Active issues under investigation
        resolved/       # Resolved issues (archived)
    problems/
        active/         # Known problems being worked on
        resolved/       # Fixed problems (archived)
    templates/          # Issue and problem templates
```

## Issues vs Problems

| Type | Definition | Examples |
|------|------------|----------|
| **Issue** | Specific bug or unexpected behavior | API timeout, validation error |
| **Problem** | Broader challenge or pattern | Architecture limitation, recurring failures |

## Naming Convention

**Pattern:** `[TYPE]-[ID]-[Short_Description].md`

**Examples:**
- `ISSUE-001-Kling_API_Timeout.md`
- `PROBLEM-001-Background_Preservation.md`
